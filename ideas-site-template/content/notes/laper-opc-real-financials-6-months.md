# OPC 公司真实财报：Laper 发布的 6 个月后

> 赵纯想，2026 年 6 月

## 一些数据

六个月后，Laper 的关键数字：

- MRR：$7,609
- 过去四周收入：$10,031
- 最低订阅：$20/月（¥140/月）
- 最高订阅：$400/月
- 退订率：18%
- 覆盖 16 个国家的专业编剧

![Laper数据](https://syd-idea.pages.dev/assets/laper/img0.jpg)

团队只有两个人：赵纯想负责产品，梁适之从零学增长。每周 5 次晨间会议，每周提交代码 200 次、发版 20 次，100% 由 Claude Code 与 CodeX 完成。

150 万行 AI 生成的代码。0 元融资。0 张 PPT。

## 现实摩擦

- Stripe 支付宝付款问题导致约 **30%** 的付款意图流失——因为 Stripe 有时会跳转到网页版支付宝并要求输入支付密码。
- 海外信用卡的 Stripe 风控问题带来的损失更严重。

从纯血 Seedance 的表现，到 DeepSeek 4.0 驱动的 Agent，Laper 完成了从专业剧作到专业影像的闭环。

## AI 务虚三杰

赵纯想、玉伯、西元，三个在杭州咖啡馆聊过天的人，六个月后各自做出了气质相近的产品：

| 产品 | 方向 | 状态 |
|---|---|---|
| Youmind | 人的全能创作欲 | 已发布 1.0 |
| Fatetell | 人的命与运 | 已发布 Brandbook |
| Laper | 人的剧作冲动 | MRR $7,609 |

三个让投资人一听就头大的命题。

![AI务虚三杰](https://syd-idea.pages.dev/assets/laper/img1.jpg)

其中最神奇的一段：西元和命理学合伙人小霖来武汉找赵纯想，在客厅做了一次 Fatetell 同款命理演绎。结果赵纯想当天卸载了 Tinder，第二天开始改睡西边卧室，废弃主卧。「Laper 的营收曲线从那一天开始抬头，真是神奇得要骂脏话了。」

## 核心判断：血盆大口，吃不停歇

这篇文章的真正洞见不在数字，而在对 Agent 生态的结构性判断。

**模型厂的桌面端产品正在加速吃掉一切 Agent 1.0 产品。**

两个典型案例：

1. **CodeX + tldraw** → 一个插件就让 CodeX 自包含了 Lovart 的同款体验，且包含在 CodeX 订阅费里。
2. **Claude + Palmier** → 让 Claude 自包含了 ChatCut（Agent + 剪辑）的体验。

![模型吞噬](https://syd-idea.pages.dev/assets/laper/img2.jpg)

模型厂桌面端产品像一张张血盆大口：

> 没有吃不掉的，只有还没吃的。CodeX、Claude Code、扣子桌面端、腾讯 WorkBuddy——只要推一个版本，无限画布有了，剪辑有了，自媒体 B-roll 制作也有了。你不能指望模型厂的桌面端产品永远只做 Code 和 PPT。

## 安全距离：极度垂直

很长一段时间，关于安全距离的共识是「要垂直」。但 Agent + 老需求（如 Agent + 剪辑）仍然不够安全，因为模型厂随时可以把这个「老需求」吞掉。

真正能活下来的路径是：

> 超垂直专家 Agentic 产品。

不是 Agent + 通用需求，而是 Agent + 一个**模型厂不愿碰也不想碰的领域**。

Laper 选了「专业编剧的剧作冲动」——这个市场不大，但足够深，且模型厂没有动力去服务一个如此狭窄的专业群体。

## {mine}

这是目前看到的最诚实的 AI 创业复盘之一。

几个点值得单独记：

1. **「Agent 产品活不了，但超垂直专家 Agentic 产品或许能活」**——这和 Studio 的处境高度相关。Studio 如果做成「通用世界创作工具」，迟早被 Claude 自带的 world building 吃掉。但如果 Studio 极度垂直——只做「让已有世界观持续生长的工作台」——它就有安全距离。

2. **2 人团队 + 100% AI 编码**——这是全新的创业范式。不是「AI 辅助人类」，而是「AI 就是工程团队」。Laper 的 150 万行代码全是 AI 生成的。这个模式对 Studio 的开发路径有直接参考意义。

3. **30% 的支付摩擦**——支付宝跳转到网页版要求输密码，就把 30% 的付费用户吓跑了。这个级别的 friction 对任何全球化订阅产品都是致命伤。中国用户出海必须解决支付体验。

4. **「AI 务虚三杰」的产品气质**——Youmind、Fatetell、Laper 都不是解决「效率问题」的产品，而是解决「人的表达冲动、命运追问、创作欲望」的产品。这种产品方向天然避开与模型厂正面竞争——因为模型厂做的是基础设施，不是精神消费品。

5. **Laper 的路线对 Studio 的参考**——Laper 从「专业编剧工具」做到「专业影像闭环」，路径是先在一个极窄群体站稳，再纵向扩展能力边界。Studio 也可以类似：先在「世界观创作者」这个极窄群体里做出不可替代性，再向 interactive story / cinematic moment 扩展。
